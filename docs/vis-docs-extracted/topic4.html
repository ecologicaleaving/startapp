VIS web service documentation
Application identifier

For each request, the client should provide an application identifier.

The identifier is unique to an application (not of an application developer). So if you develop more than one application that uses the FIVB web service, please request one identifier for each application.

This page contains the following information:

  • Getting an identifier
  • Using the identifier in a request
  • Getting the web service log for the application
 Getting an identifier

To get an application identifier, you must:

  1. Have a developer account on the FIVB.
  2. Request an application identifier for your application.

Currently, the only way to create a developer account and and request an application identifier is to contact the FIVB using the following e-mail address: vis.sdk@fivb.org.

Later, we will add web pages, so that you can directly create an account, display your applications and view and manage the web service log for your application.

 Using the identifier in a request

To add the application identifier in the request, you must use the X-FIVB-App-ID custom header:

Application identifier header
Copy Code 
X-FIVB-App-ID: <identifier>
