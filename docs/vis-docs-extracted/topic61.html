Volleyball Tournament Ranking Entry

Represents an entry in the ranking for a volleyball tournament.

The name of the XML element for an entry for the ranking is <VolleyballTournamentRanking>.

 Properties

The following table lists all the properties of an entry for the ranking for a volleyball tournament.

Name Type Description
DeletedDT DateTime Empty if the ranking entry is valid.
Date and time of the deletion when the entry is deleted.
LastChangeDT DateTime The date and time of the last change of the ranking entry.
LastChangeUser Int32 The number of the user who has made the last change of the ranking entry.
LastChangeUsername String The name of the user who has made the last change of the ranking entry.
No Int32 The unique number of the ranking entry.
NoTeam Int32

The unique number of the team.
This property can be used to retrieve the information about the team in the VolleyTeam data.
The value is empty if the team is unknown. In this case, you can use the TeamSource property for a description of the team.
NoTournament Int32 The unique number of the tournament.This property can be used to retrieve the information about the tournament in the VolleyTournament data.
Position Int32 The position of the team in the ranking.
Is always from 1 to the number of teams in the tournament.
This field must be used to sort the ranking in the right order.
Rank Int32

The rank.
When there is a tie between two or more teams, they can have the same rank. In this case, the first team has a positive ranking and the other ones have a negative ranking with the same absolute value. The negative rank values should not be displayed to the user.
RankText String The text to display for the rank.
For tied teams, the value is the rank for the first team and an empty string for the other ones.
RankTextWithRepeat String The text to display for the rank, repeating the rank.
For tied teams, the value is the rank for all teams.
This property is used when the ranking is displayed in a list that can be filtered or grouped, so that the rank is always visible.
TeamCode String The code of the team.
If you need to display always something for the team, you should use the TeamNameOrCodeOrSource property.
TeamName String The name of the team.
If the team is not known (field NoTeam is empty), the value of this property is empty.
If you need to display always something for the team, you should use the TeamNameOrCodeOrSource property.
TeamNameOrCodeOrSource String The name, code or source of the team, using the first defined property.
Should be used to display something for the team.
TeamSource VolleyTournamentRankingTeamSource The source of the team.
This attribute contains an XML element that describes how the team is selected.
If you need to display always something for the team, you should use the TeamNameOrCodeOrSource property.
Version Int32 Version of the ranking entry.
The version is global to all ranking entries. It is used when requesting a list of ranking entries, to get only the changes of the list.
