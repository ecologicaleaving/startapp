User access - Parameters

When defining conditions, you can use parameters. In this case, the value of the property is replace by a parameter place holder: {name}, where name is the name of the parameter.

The following parameters are taken from the user information:

Name Value
NoUser The number of the user.
Always defined.
UserType The type of the user.
Always defined.
UserTypeNo The number related to the type of the user.
For example, if the user is for a player, this is the number of the player.
Always defined.
ConfederationCode The code of the confederation.
If only the federation is defined, the web service will take the confederation to which the federation belongs.
Can be undefined.
NoConfederation The number of the confederation.
The web service calculates it automatically from the code of the confederation, if defined.
Can be undefined.
FederationCode The code of the federation.
Can be undefined.
NoFederation The number of the federation.
The web service calculates it automatically from the code of the federation, if defined.
Can be undefined.

The parameters are only used when defining the conditions. When the authorizations are sent to a client, they are replaced by the value of the parameter.
