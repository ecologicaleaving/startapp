VBA statistics file

This is a preliminary version of the documentation and there can remain errors on it.

If you find errors (even typos) in this documentation, please contact the FIVB.

The VBA statistics file is named Statxxx.i, where xxx is the number of the match in the tournament, using 3 digits, and i is 1 for the first acquisition PC and 2 for the second one.

It exists in different formats, for 20 or 25 players. Unfortunately, there is no way to detect the version of the file.

The content of the file is a collection of entries. The available entries can be identified by bits 0 to 4 of their first byte. Normally, these bits contain the shirt number of the player, but can contain special values to identify the following data.

The different entries for the file are listed in the following sections.

 Action

An action is packed into a 16-bits integer, with the LSB first.

The format of the packed integer is the following:

  • Bits 0 to 4 (mask is 0x001F) contain the shirt number. It can be 1 to 20 or 1 to 25, depending on the format. Some special values are defined, see the table below for them.
  • Bits 5 to 9 (mask is 0x03E0) contain the skill, see the table below for the values.
  • Bits 10 to 14 (mask is 0x7C00) contain the note, see the table below for the values.
  • Bits 15 is 0 for team A or 1 for team B.

 End of set (version 1)

The version 1 is used only in the 20 players format. It is coded with two 16-bits values, starting with the LSB.

The first value is:

  • Bits 0 to 4 (mask is 0x001F) contain the special code: 30 (0x001E).
  • Bits 5 to 9 (mask is 0x03E0) contain the number of the set.

The second value is:

  • Bits 5 to 9 (mask is 0x03E0) contain the number of points of team A.
  • Bits 10 to 14 (mask is 0x7C00) contain the number of points of team B.
 End of set (version 2)

An end of set is coded with two 16-bits values, starting with the LSB.

The first value is:

  • Bits 0 to 4 (mask is 0x001F) contain the special code: 26 (0x001A) for 20 players format or 28 (0x001C) for 25 players format.
  • Bits 5 to 9 (mask is 0x03E0) contain the number of the set.

The second value is:

  • Bits 0 to 7 (mask is 0x00FF) contain the number of points of team A.
  • Bits 8 to 15 (mask is 0xFF00) contain the number of points of team B.
 Libero substitution

Libero substitution is a variant of player substitution. It is used when the player leaving the court is a libero.

  • Bits 0 to 4 (mask is 0x001F) contain the special code: 24 (0x0018) for 20 players format or 26 (0x001A) for 25 players format.
  • Bits 5 to 9 (mask is 0x03E0) contain the shirt number of the libero leaving the court.
  • Bits 10 to 14 (mask is 0x7C00) contain the shirt number of the libero entering the court.
  • Bit 15 is 0 for team A or 1 for team B.
 New line-up

A new line-up is coded with a 16-bits header (starting with the LSB) and six 8-bits value.

The header is a special value:

The data contains the shirt numbers of the 6 players, starting with the server (bottom right) and going counterclockwise.

 Player substitution
  • Bits 0 to 4 (mask is 0x001F) contain the special code: 28 (0x001C) for 20 players format or 29 (0x001D) for 25 players format.
  • Bits 5 to 9 (mask is 0x03E0) contain the shirt number of the player leaving the court.
  • Bits 10 to 14 (mask is 0x7C00) contain the shirt number of the player entering the court.
  • Bit 15 is 0 for team A or 1 for team B.
