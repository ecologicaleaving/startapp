Request to get a list of volleyball tournaments
 Request

The syntax for this request is:

Web service request
Copy Code 
<Request Type="GetVolleyTournamentList"
         Fields="<list of the fields to return>">
         Version="<version of local list>">
  <Filter /> <!-- optional: contains the filter to use -->
</Request>

Fields

The list of fields is mandatory.
It can contain all the fields in the VolleyTournament data. Only the fields you have access to will be returned.

Filter

The filter is optional.
If it is not specified, the response will contain all the beach volleyball tournaments.
Please see the VolleyTournamentFilter description for more information.

Version

The version is optional.
The goal of this parameter is to request an update to an existing local list: for the first request, you don't specify the parameter (or set it to 0) and you receive the full list of the filtered tournaments; for subsequent requests, you specify the version of the local list and the web service only returns the changes (new tournaments or tournaments that have changed). Doing this will decrease the bandwidth used by the local application.
Note that the version of all the tournaments (not only the filtered ones) is returned in the root of the response.

 Response

When the request is successful, the web service returns the filtered list of volleyball tournaments.

In addition, the root element contains the number of tournaments that are returned and the version of all the tournaments.

 Security

This is a public request: any client can retrieve information.

Some of the fields for a volleyball tournament are public and other are not.
The response from the web service will contain only the fields the client has access to.

 Example

Get the list of the tournaments confirmed, running and finished for the 2017 season.

Web service request
Copy Code 
<Request Type="GetVolleyTournamentList"
         Fields="No Code Name StartDate EndDate">
  <Filter Seasons="2017" Statuses="Scheduled Running Finished"/>
</Request>
