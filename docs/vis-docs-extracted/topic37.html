Beach Volleyball World Tour Ranking Entry

Represents an entry in the beach volleyball World Tour Ranking.

The name of the XML element for an entry for the ranking is <BeachWorldTourRankingEntry>.

 Fields

The following table lists all the fields in an entry for the beach volleyball World Tour Ranking.

Name Type Description
EarnedPointsPlayer Int32 Earned points for each player.
EarnedPointsTeam Int32 Earned points for the team.
This is 2 times EarnedPointsPlayer.
EarningsBonusPlayer Amount Bonus earnings for each player, in US$.
If the player has no bonus earnings, this attribute can be omitted.
EarningsBonusTeam Amount Bonus earnings for the team, in US$.
This is 2 times EarningsBonusPlayer.
If the team has no bonus earnings, this attribute can be omitted.
EarningsPlayer Amount Earnings for the each player, in US$.
If the player has no earnings, this attribute can be omitted.
EarningsTeam Amount Earnings for the team, in US$.
This is 2 times EarningsPlayer.
If the team has no earnings, this attribute can be omitted.
EarningsTotalPlayer Amount Total earnings for each player, in US$.
This is the total of EarningsPlayer and EarningsBonusPlayer.
If the player has no earnings, this attribute can be omitted.
EarningsTotalTeam Amount Total earnings for the team, in US$.
This is the total of EarningsTeam and EarningsBonusTeam, or 2 times EarningsTotalPlayer.
If the team has no earnings, this attribute can be omitted.
HasTournamentsRemoved Boolean

Indicates whether some tournaments have been removed from the teams results for the ranking.
If the value is false, this attribute can be omitted.

The number of tournaments to take into account for the ranking for each season is limited.
When a team has played more tournaments than this number, this field is set.
NbParticipations Int32 Number of participations in the tournaments during the ranking period.
NoPlayer1 Int32 Unique number of the player 1.
This field can be used to retrieve the information about the player in the Player data.
NoPlayer2 Int32 Unique number of the player 2.
This field can be used to retrieve the information about the player in the Player data.
Position Int32 Position of the team in the ranking.
Is always from 1 to the number of teams in the ranking.
This field must be used to sort the ranking in the right order.
Rank Int32 Rank of the team.
When there is a tie between two or more teams, they can have the same rank. In this case, the first team has a positive ranking and the other ones have a negative ranking with the same absolute value. The negative rank values should not be displayed to the user.
TeamFederationCode FederationCode Code of the federation for the team.
TeamName String Name of the team.
