Volleyball Players Ranking

Represent the players' ranking of a volleyball tournament.

The rankings are per skill.
The name of the XML element for a ranking is <Ranking>.
The name of the XML element for a list of rankings is Rankings>.

The data returned for a ranking depends on the skill.

This page contains the following information:

  • Common properties
  • Properties for non-scoring skills (Dig, Libero, Reception and Set)
  • Properties for scorers
  • Properties for scoring skills (Block, Serve and Spike)
  • Rank and position
 Common properties

The following properties are available for all the rankings.

Name Type Description
FirstName String The first name.
LastName String The last name.
NbMatches Int32 The number of matches played.
NbSets Int32 The number of sets played.
No Int32 The number of the player.
NoShirt Int32 The shirt number
NoTeam Int32 The number of the team.
Position Int32 The position in the ranking.
The value is 1 for the best player and counts downward.
Rank Int32 The rank team.
When there is a tie between two or more players, they can have the same rank. In this case, the first player has a positive ranking and the other ones have a negative ranking with the same absolute value. The negative rank values should not be displayed to the user.
TeamCode String The code of the team.
 Properties for non-scoring skills (Dig, Libero, Reception and Set)

The following properties are available for the non-scoring skills, in addition to the common properties.

Name Type Description
Continues Int32 The number of attempts with the rally continuing.
Excellents Int32 The number of excellent gestures.
Faults Int32 The number of faults.
Note String The note.
The property is only returned for players that are ranked and have a note.
TotalAttempts Int32 The total number of attempts.
This is the sum of Continues, Excellents and Faults.
 Properties for scoring skills (Block, Serve and Spike)

The following properties are available for scoring skills, in addition to the common properties.

Name Type Description
Continues Int32 The number of attempts with the rally continuing.
Faults Int32 The number of faults.
Note String The note.
The property is only returned for players that are ranked and have a note.
Points Int32 The number of points
TotalAttempts Int32 The total number of attempts.
This is the sum of Continues, Faults and Points.
 Properties for scorers

The following properties are available for scorers, in addition to the common properties.

Name Type Description
Blocks Int32 The number of points scored with blocks.
Points Int32 The total number of scored points.
Services Int32 The number of points scored with services.
Spikes Int32 The number of points scored with spikes.
 Rank and Position

The Rank property con be negative for tied players: the first player has a positive value and the tied ones have a negative value of the same absolute value.

As the Rank property can be negative, you must not use it for sorting, instead, you must use the Position property, that always starts at 1 and is incremented for each player.
