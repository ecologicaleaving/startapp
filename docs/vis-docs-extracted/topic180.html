Request to save a player
 Request

The syntax for this request is:

Web service request
Copy Code 
<Request Type="SavePlayer">
  <Player No="<player number>"
          <!-- List of fields to insert and / or change -->
          />
</Request>

Player number

The No attribute is optional:

  • If it is specified, the player with the specified number will be updated.
  • If it is not specified, a new player will be inserted. In this case the number of the inserted player will be returned by the service.

Fields

You can specify any player field in the request, but only the fields you have the right to insert or change are used by the service.

 Response

For an insertion, the response will contain the number of the new player:

Web service response
Copy Code 
<Player No="<number of the new player>" />

For a change, the result will be simply a success indication:

Web service response
Copy Code 
<OK />
 Errors

In addition to global errors, the following errors can be sent by the server:

Error Description
<BadParameter> The No parameter is specified and is not a valid 32-bits integer number.
<NoData> Update: there is no player with the specified number.
 Security

This is a private request: only authorized clients can change or update a player.

For insertion, the authorized clients can have limitations on the fields they can insert and, in some cases, to the values of some fields.
For updating, the authorized clients can have limitations on the fields they can change and, in some cases, to the values of some fields.
In both cases, the service will not complain if the client can not insert or change a field: the new field value will be simply ignored.

 Example - Insert

Insert a new player:

Web service request
Copy Code 
<Request Type="SavePlayer">
  <Player FirstName="John"
          LastName="Smith"
          Nationality="US"
          Federation="USA" />
</Request>
 Example - Update

Change the last name of player number 100002:

Web service request
Copy Code 
<Request Type="SavePlayer">
  <Player No="100002"
          LastName="Smith" />
</Request>
 Remarks
If the player is related to a user in the system, the user has never do a login and the e-mail address of the player is modified, the e-mail of the user will also be updated, a new password will be generated for him/her and an e-mail will be sent to him/her with his/her username and new password.
This is done to have a possibility to change the e-mail address of an automatically generated user if the previous e-mail address was not right.
The newly generated password avoids for someone else to change the e-mail address just to retrieve the password and correct it later.
