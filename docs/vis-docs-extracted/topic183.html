Request to upload volleyball live data

This request is used when live data must be uploaded to the web service.

 Request

The syntax for the request is:

Web service request
Copy Code 
<Request Type="UploadVolleyLive"
         ClientId="<client identifier>"
         ClientRole="<computer role>"
         NoMatch="<match number>">
  <VolleyLive ... />
</Request>

Where:

Attribute Description
ClientId The identifier of the client.
This is a UUID.
For a single match, all the clients must have a unique identifier.
This is a mandatory attribute.
ClientRole The role of the client.
This is a mandatory attribute.
NoMatch The number of the match.
This is a mandatory attribute.

VolleyLive is the live data to upload.

 Response

When the request is successful, the web service returns an <OK> XML element.

Web service response
Copy Code 
<OK />
 Errors

In addition to global errors, the following errors can be sent by the web service:

Error Description
<AccessDenied> You don't have the right to upload live data for the match.
<BadParameter> - The IdClient attribute is not a valid UUID.
- The Role attribute is not a valid role.
- There is one or more errors in the live data. See the element content for more information about the error.
<NoData> The match doesn't exist.
<ParameterMissing> - The IdClient attribute is missing.
- The Role attribute is missing.
- The VolleyLive children element is missing.

For the <BadParameter> error, the following tests are made by the web service:

  • Each value has the right format for its type.
  • The NoServingTeam attribute of <VolleyLive> corresponds to a team of the match.
  • The <LineUp> child element of <VolleyLive> occurs at most twice.
  • The NoTeam attribute of <LineUp> corresponds to a team of the match.
  • The player's numbers attribute of <LineUp> corresponds to the players of the right team
  • The player's numbers attribute of <LineUp> corresponds to the players registered for the team in the tournament
  • The No attribute of <Match> is specified and corresponds to an existing match.
  • The <Team> element occurs at most twice.
  • The No attribute of <Team> is specified and corresponds to a team of the match.
  • The No attribute of <Team> is unique.
  • The No attribute of <Player> is specified and correponds to a player registered for the team in the tournament.
  • The No attribute of <Player> is unique.
  • The No attribute of <Set> is specified and is between 1 and 5.
  • The No attribute of <Set> is unique.
  • The NoServingTeam attribute of <Set> corresponds to a team of the match.
  • The scores of the match and the sets are valid.
 Security

This is a private request: a username and password is required. Otherwise, an <AccessDenied> XML element is returned by the web service.

A user can be limited to upload live data only for some matches.
