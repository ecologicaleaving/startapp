Volleyball Pool Ranking Entry

Represents an entry in the ranking for a volleyball pool.

The name of the XML element for an entry for the ranking is <VolleyballPoolRanking>.

This page contains the following information:

  • Matches by result
  • Points and Sets ratios
  • Properties
  • Rank and Position
 Properties

If the team has never played a match (both points won and points lost are 0), the value is -2 and should be displayed to the user as an empty string.
If the points lost are 0, the value is -1 and should be displayed to the user as <MAX>.
Otherwise, it is 1'000 times the value that must be displayed. The value must always be displayed with 3 digits after the decimal separator.
PointsRatioText String The points ratio, correctly formatted as a text.
PointsWon Int32 The total number of points won.
Position Int32

The position in the ranking.
Is always from 1 to the number of teams in the tournament.
This field must be used to sort the ranking in the right order.

As the GetVolleyPoolRanking request always sorts the ranking using the position, there is normally no need to get this value.
Rank Int32

The rank.
When there is a tie between two or more teams, they can have the same rank. In this case, the first team has a positive ranking and the other ones have a negative ranking with the same absolute value. The negative rank values should not be displayed to the user.
RankText String The text to display for the rank.
For tied teams, the value is the rank for the first team and an empty string for the other ones.
RankTextWithRepeat String The text to display for the rank, repeating the rank.
For tied teams, the value is the rank for all teams.
This property is used when the ranking is displayed in a list that can be filtered or grouped, so that the rank is always visible.
SetsLost Int32 The total number of sets lost.
SetsRatio Int32 The sets ratio.
If the team has never played a match (both sets won and sets lost are 0), the value is -2 and should be displayed to the user as an empty string.
If the sets lost are 0, the value is -1 and should be displayed to the user as <MAX>.
Otherwise, it is 1'000 times the value that must be displayed. The value must always be displayed with 3 digits after the decimal separator.
SetsRatioText String The sets ratio, correctly formatted as a text.
SetsWon Int32 The total number of sets won.
TeamCode String The code of the team.
TeamName String The name of the team.
Version Int32 Version of the ranking entry.
The version is global to all ranking entries. It is used when requesting a list of ranking entries, to get only the changes of the list.
 Properties by match format

Matches in a pool can be played in 5 sets or in 7 sets. You can get the number of sets by looking at the format of a match in the pool. There are two sets of properties for the matches by result, one for matches in 5 sets and one for matches in 7 sets. If you need this information, you must get the right properties, depending on the matches format.

Matches in 5 sets

You must use the following properties:

  • MatchesAt03
  • MatchesAt13
  • MatchesAt23
  • MatchesAt30
  • MatchesAt31
  • MatchesAt32

Matches in 7 sets

You must use the following properties:

  • MatchesAt04
  • MatchesAt14
  • MatchesAt24
  • MatchesAt34
  • MatchesAt40
  • MatchesAt41
  • MatchesAt42
  • MatchesAt43
 Rank and Position

The Rank property con be negative for tied teams: the first team has a positive value and the tied ones have a negative value of the same absolute value. To display the rank correctly, you can use the RankText property, that is empty for tied teams. If the ranking you're displaying can be filtered or grouped, you can use the RankTextWithRepeat property, that has always a rank.

As the Rank property can be negative, you must not use it for sorting, instead, you must use the Position property, that always starts at 1 and is incremented for each team.

 Points and sets ratios

When a team has not played any match, its ratios cannot be calculated and should be displayed as an empty string. The value in the ratios properties is -2.

When a team has played matches, but has not lost any point or set, its corresponding ratio is infinite and should be displayed as <MAX>. The value in the corresponding ratio property is -1.

You should use the PointsRatioText and SetsRatioText property to display the ratios correctly.
