User access - Conditions

Conditions are optional for fields and permissions. They indicate if a field or action is available only if some conditions are met.

 Syntax

The syntax for a condition is the following:

Syntax
Copy Code 
<Property name> <Operator> <Value>

Where:

  • <Property Name> is the name of a property.
  • <Operator> is the operator: = for equality or <> for inequality.
  • <Value> is the value of the property. For a discrete value, it must be its name.

The property name can a the name of a single property, or a list of properties separated by a dot, for example:

  • FirstName: the first name of the player.
  • Federation.Code:

the code of the federation of the player.

For JSON output, the properties names are converted to camel case.

 Use

The conditions must be defined inside the field or permission definition:

Limit the edition of the first name of a player
Copy Code 
<Field Name="FirstName">
  <Condition>Status="Draft"</Condition>
  <Condition>FederationCode=SUI</Condition>
</Filed>

The first name of the player can only be edited if his/her status is Draft and the code of his/her federation is SUI.

If there are more than one condition and the any of the condition permits the field or the action, the entry must be repeated:

Limit the edition of the first name of a player
Copy Code 
<Field Name="FirstName" Condition="Status=Draft" />
<Field Name="FirstName" Condition="FederationCode=SUI" />

The first name of the player can only be edited if his/her status is Draft or the code of his/her federation is SUI.

If there is a single condition on a field, the condition can be in the field definition:

Limit the edition of the first name of a player
Copy Code 
<Field Name="FirstName" Condition="Status=Draft" />

The first name of the player can only be edited if his/her status is Draft.

 Definition

For the definition of a condition, in addition to values, you can also use parameters:

Defining a condition
Copy Code 
<Field Name="FirstName" Condition="FederationCode={FederationCode}" />

The first name of the player can only be edited if the code of his/her federation is the federation of the logged in user.

See Parameters for a list of the available parameters.
