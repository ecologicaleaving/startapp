Filter for press releases

Filter for press releases.

 Attributes

The following table lists the attributes for the filter for press releases.

Name Type Description
Accepted Boolean Accepted state for the press release texts.
The meaning of the values is:
Value Description

Empty
OR
Not specified

 The web service will return all the press releases, not taking account of the accepted state of their texts.
True The web service will return only the press release whose at least one of their text is accepted.
False The web service will return only the press release whose at least one of their texts is not accepted.

If this attribute is specified, the Languages property of the returned press releases will contain only the languages for the texts that are accepted or not, depending of the value of this attribute.
Categories PressReleaseCategory Categories for the press releases.
Each category must be specified by its name or value, and, if more than one category is specified, they must be separated with spaces.
If it is not specified, all categories are included in the returned press releases.
PublishOn PressReleasePublishOn Where the press release should be published.
Each location must be specified by its name or value, and, if more than one location is specified, they must be separated with spaces.
If it is not specified, the returned press releases are not filtered by this criteria.
SpecialTypes PressReleaseSpecialType Types for the press releases.
Each type must be specified by its name or value, and, if more than one type is specified, they must be separated with spaces.
If it is not specified, the returned press releases are not filtered by this criteria.
TournamentCodes String Code of the tournaments related to the press release.
If more than one code is specified, they must be separated with spaces.
If it is not specified, the returned press releases are not filtered by tournament code.
