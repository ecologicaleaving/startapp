VIS web service documentation
Identification

The identification step is necessary for all clients that need to send data into the volleyball match currently played. It permits to identify who is connected and to assign them the port(s) to use to send the data.

 Port
By default, the identification is made using TCP port 52915 on the match management PC. This is the default port number, but it can be changed on the VIS application. So you must make an option on the client application to change the port number, in addition to the username and password.
 Identification request

The request is made using an XML element.

Here is an example of an identification request for the video acquisition client:

Video acquisition client
Copy Code 
<Identification NoMatch="123" Username="my Username" Password="my Password" Role="VideoTracking" Version="1" />
 Answer

The answer is an XML element returned by the match management PC.

If the identification is successful, the answer contains the number of the port to use for future communications with the match management PC. The port is tied to the IP address of the PC that has asked for identification. If another PC tries to connect to the port, it will be rejected.
The port is open for 1 minute. If no authorized PC is connected during this time, the port will be closed and the PC must identify itself again.

Successful request
Copy Code 
<Accepted NoPort="123" />

If the Match attribute has not a valid value, the following XML will be sent:

Error: bad match number
Copy Code 
<BadMatchNo />

If the Role attribute has not a valid value, the following XML will be sent:

Error: bad role
Copy Code 
<BadRole />

If the username and/or password are not correct, the following XML element will be sent:

Error: bad username and/or password
Copy Code 
<BadUsernameOrPassword />

If the Version attribute has a value that is not supported by the match management software, the following XML element will be sent.

Error: version not supported
Copy Code 
<VersionNotSupported />
