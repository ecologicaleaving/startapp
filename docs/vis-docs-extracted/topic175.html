Request to get a list of articles
 Request

The syntax for this request is:

Web service request
Copy Code 
<Request Type="GetArticleList"
         Fields="<list of the fields to return>">
         Version="<version of local list>">
  <Filter /> <!-- optional: contains the filter to use -->
  <Relation Name="<name of the relation>" Fields="<list of the fields for the relation>">/> <!-- optional: contains a relation (can be used multiple times) -->
</Request>

Fields

The list of fields is mandatory.
It can contain all the fields in the Article data. Only the fields you have access to will be returned.

Filter

The filter is optional.
If it is not specified, the response will contain all the articles.
Please see the ArticleFilter documentation for more information.

Relations

The relations are optional. They permit to get additional information about the article, such as the texts or the tags.
A list of the available relations for an article can be found here.

Version

The version is optional.
The goal of this parameter is to request an update to an existing local list: for the first request, you don't specify the parameter (or set it to 0) and you receive the full list of the filtered articles; for subsequent requests, you specify the version of the local list and the web service only returns the changes (new articles or articles that have changed). Doing this will decrease the bandwidth used by the local application.
Note that the version of all the articles (not only the filtered ones) is returned in the root of the response.

 Response

When the request is successful, the web service returns the filtered list of articles.

In addition, the root element contains the number of articles that are returned and the version of all the articles.

 Security

This is a public request: any client can retrieve information.

Some of the fields for an article are public and other are not.
The response from the web service will contain only the fields the client has access to.

 Example

Get the list of the articles related to player number 110419.

Web service request
Copy Code 
<Request Type="GetArticleList"
         Fields="No DateTime Source">
  <Filter>
    <Tags>
      <Tag>Player:110419</Tag>
    </Tags>
 </Filter>
  <Relation Name="Tags" Fields="TagText TagUserText" />
  <Relation Name="Texts" Fields="LanguageCode LanguageEnglishName LanguageNativeName AuthorName Headline Introduction Text" />
</Request>

The response is:

  • XML
  • JSON
