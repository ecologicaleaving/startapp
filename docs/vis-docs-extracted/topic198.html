Request to delete a volleyball transfer
 Request

The syntax for this request is:

Web service request
Copy Code 
<Request Type="DeleteVolleyTransfer"
         No="<transfer number>" />

Where:

Attribute Description
No Number of the transfer to delete. This is a mandatory parameter.
 Response

When the request is successful, the the service will return the following response:

Web service response
Copy Code 
<OK />
 Errors

In addition to global errors, the following errors can be sent by the server:

Error Description
<AccessDenied> The user does not have the right to delete this volleyball transfer.
<BadParameter> The No parameter is not a valid 32-bits integer number.
<BadStatus> The transfer has not the Draft or Pending status.
<NoData> The transfer with the specified number does not exist or has been deleted.
<ParameterMissing> You must specify the No parameter.
 Security

A public client does not have access to this request: it will receive an <AccessDenied> answer.

A private client must have the CanDelete permission for the transfer he/she wants to delete.
