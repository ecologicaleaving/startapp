Request to upload a document for a beach volleyball match

This request is used when a document for a beach volleyball match must be uploaded.
If the document is a special document (for example the scoresheet, you should use the UploadSpecialDocument request.

When receiving this request, the web service will save the document and return the number of the document.

 Request

The syntax for this request is:

Web service request
Copy Code 
<Request Type="UploadBeachMatchDocument"
         Description="<description>"
         Document="<document>"
         DocumentType="<document type>"
         File="<filename>"
         FileNameExtension="<extension of the file name>"
         NoMatch="<number of the match>" />

Where:

Attribute Description
Description The description of the document.
This is an optional attribute.
Document The document.
The document must be encoded using Base64 encoding.
This is a mandatory attribute.
DocumentType The document type.
This attribute can contain the value or the name of the type.
This is an optional attribute, if it is not specified, the document type will be Other.
FileName The name of the file, without extension.
This is a mandatory attribute.
FileNameExtension The extension of the file name.
This attribute is used to determine which kind of document file is sent.
This is a mandatory attribute.
NoMatch The unique number of the match.
This is a mandatory attribute.
 Response

When the request is successful, the service will return an <OK> XML element, with the number of the document:

Web service response
Copy Code 
<OK No="xxxx" />
 Errors

In addition to global errors, the following errors can be sent by the web service:

Error Description
<AccessDenied> You don't have the right to upload a special document for the specified item.
<BadParameter> - The Document attribute is not correctly encoded.
- The DocumentType attribute has not a valid value.
- The NoMatch attribute is not a valid integer number.
<NoData> The item related to the document doesn't exist.
<ParameterMissing> - The Document attribute is missing.
- The DocumentType attribute is missing.
- The FileName attribute is missing.
- The FileNameExtension attribute is missing.
- The NoMatch attribute is missing.
 Security

This is a private request: a username and password is required. Otherwise, an <AccessDenied> XML element will be returned by the web service.

A user can be limited to upload a document only for some matches.

 Example

Upload a beach volleyball scoresheet.

Web service request
Copy Code 
<Request Type="UploadBeachMatchDocument"
         Description="Log file for acquisition"
         Document="XXXX"
         FileNameExtension="log"
         NoMatch="1234" />

The response is (formatted for readability):

Web service response
Copy Code 
<OK No="201512345" />
