Volleyball Match Acquisition Method

The method used to get live data for a volleyball match.

Depending on the method used for acquisition, the web service must made arrangements on where the data is and how it must be processed to serve live data to clients. This value is used internally to process live data and return live data always in the same format.

The following table lists the values for the acquisition methods for a volleyball match.

Value

 Name Description

1

 Vba_Ruby The acquisition is made using the VBA method, with the Ruby script converting the data,

2

 DataVolley_Vba_Ruby The acquisition is made using the DataVolley application that send information compatible with VBA, with the Ruby script converting the data.

3

 Vis The acquisition is made with the VIS method.

4

 Vba_Vis The acquisition is made using the VBA method, with the VIS web service converting the data,

5

 DataVolley_Vba_Vis The acquisition is made using the DataVolley application that send information compatible with VBA.

6

 DataVolley_Vis The acquisition is made using the DataVolley application that send information compatible with VIS.

7

 OG2016

The acquisition is made using the 2016 Olympic Games method.
This is special format that is used only in the Olympic Games 2016, with the following characteristics:

  • The score and the rosters come from the e-Scoresheet application.
  • The statistics come from the OG data, converted to VBA format.
  • We only have the statistics for the match, not for the individual sets.

8

 None There is no acquisition of live data for the match.
