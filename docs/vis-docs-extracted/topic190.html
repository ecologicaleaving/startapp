Request to get a volleyball pool
 Request

The syntax for this request is:

Web service request
Copy Code 
<Request Type="GetVolleyPool"
         No="<pool number>">
         Fields="<Optional: list of the fields to return>" />

Where:

Attribute Description
No The number of the pool.
This is a mandatory parameter.

Fields

The list of fields is optional: if it not specified, all the fields you have access to will be sent.
It can contain all the fields in the VolleyPool data. Only the fields you have access to will be returned.

 Response
When the request is successful, the service will return the data of a volleyball pool.
 Errors

In addition to global errors, the following errors can be sent by the server:

Code Description
BadParameter The No parameter is not a valid 32-bits integer number.
NoData The pool with the specified number does not exist or has been deleted.
ParameterMissing You must specify the No parameter.
 Security

This is a public request: any client can retrieve information.

Some of the fields for a pool are public and other are not.
The response from the web service will contain only the fields the client has access to.

 Example

Get the pool with the number 2013.

Web service request
Copy Code 
<Request Type="GetVolleyPool"
         No="2013"
         Fields="No Code Name Type">
  <Relation Name="Tournament" Fields="Code Name WebSite" />
</Request>

The response is (formatted for readability):

  • XML
  • JSON
<VolleyballPool No="2013" Code="A1" Name="Pool A1" Type="1">
  <Tournament Code="WL2017-1" Name="World League 2017 - Group 1" WebSite="https://worldleague.2017.fivb.com" No="979" Version="853"/>
</VolleyballPool>

{
  "data": {
    "no": 2013,
    "code": "A1",
    "name": "Pool A1",
    "type": 1,
    "tournament": {
      "code": "WL2017-1",
      "name": "World League 2017 - Group 1",
      "webSite": "https://worldleague.2017.fivb.com"
    }
  }
}
