User access - Values

Some fields can have limited values for specific users. For example, a user of a federation can add a new player, but only for his/her own federation.

 Use

The values must be defined inside the field definition:

Values
Copy Code 
<Field Name="FieldName">
  <Value IsDefault="True">Value1</Value>
  <Value>Value2,Value3</Value>
</Field>

The above example defines 3 values for the FieldName field: Value1, Value2 and Value3. In addition, it defines Value1 to be the default value.

For a client application that permits the user to modify the item: it should only let the user choose between the 3 defined values and it should automatically select the Value1 value, as it is the default value. If another value is set for the field, the field cannot be changed by the user.

When saving the item on the web service: if the value of the field is not one of the specified value, the change of this field is ignored.

 Definition

For the definition of a value, in addition to values, you can also use parameters:

Values definition
Copy Code 
<Field Name="FederationCode">
  <Value IsDefault="True">{FederationCode}</Value>
</Field>

The FederationCode field can only have the code of the federation of the current user.
