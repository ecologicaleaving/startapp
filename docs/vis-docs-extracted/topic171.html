Request to upload a special document

This request is used when a special document must be uploaded.

When receiving this request, the web service will save the document, update the related item with the number of the document and return the number of the document. The item is identified by the document type and the number of the item. For example, if the document is a scoresheet for a volleyball match, the caller must specify the unique number of the match. If there already exists a special document of the same type for the item, the previous document will be removed.

A special document is a document related to an item that must be identified by the VIS system. Example of these documents are the match scoresheets, a person's passport, a filled FIVB form, etc.

 Request

The syntax for this request is:

Web service request
Copy Code 
<Request Type="UploadSpecialDocument"
         Document="<document>"
         DocumentType="<special document type>"
         FileNameExtension="<extension of the file name>"
         NoItem="<number of the item related to the document>" />

Where:

Attribute Description
Document The document.
The document must be encoded using Base64 encoding.
This is a mandatory attribute.
DocumentType The special document type.
This attribute can contain the value or the name of the type.
This is a mandatory attribute.
FileNameExtension The extension of the file name.
This attribute is used to determine which kind of document file is sent.
This is a mandatory attribute.
NoItem The number of the item related to the document.
For example, for a volleyball match scoresheet, this is the unique number of the match.
This is a mandatory attribute.
 Response

When the request is successful, the service will return an <OK> XML element, with the number of the document:

Web service response
Copy Code 
<OK No="xxxx" />
 Errors

In addition to global errors, the following errors can be sent by the web service:

Error Description
<AccessDenied> You don't have the right to upload a special document for the specified item.
<BadParameter> - The Document attribute is not correctly encoded.
- The DocumentType attribute has not a valid value.
- The NoItem attribute is not a valid integer number.
<NoData> The item related to the document doesn't exist.
<ParameterMissing> - The Document attribute is missing.
- The DocumentType attribute is missing.
- The FileNameExtension attribute is missing.
- The NoItem attribute is missing.
 Security

This is a private request: a username and password is required. otherwise, an <AccessDenied> XML element will be returned by the web service.

A user can be limited to upload a special document to only some items.

 Example

Upload a beach volleyball scoresheet.

Web service request
Copy Code 
<Request Type="UploadSpecialDocument"
         Document="XXXX"
         DocumentType="BeachTournamentScoresheet"
         FileNameExtension="pdf"
         NoItem="1234" />

The response is (formatted for readability):

Web service response
Copy Code 
<OK No="201512345" />
