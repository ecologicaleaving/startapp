Match file for WinVis32 (Matchxxx.mtc)

This is a preliminary version of the documentation and there can remain errors on it.

If you find errors (even typos) in this documentation, please contact the FIVB.

This file is used to transfer match data between WinVis32 and VisAsp. It can contain the rally information of the match.

The xxx in the file name refers to the number of the match in the tournament, using 3 digits.

The file contains binary data and there exists 3 formats for this file, depending on the maximum supported shirt number:

  • For shirt numbers up to 20.
  • For shirt numbers up to 25.
  • For shirt numbers up to 99.

The main part of the file is the same for the 3 formats, only the actions coding changes.

The file contains a 64-bits header, followed by tagged data. A tag is a 16-bits integer that indicates which data follows.

 Action

An action is packed into a 16-bits integer, starting with the LSB. The value is splitted into three parts and the meaning of these parts depend on the content.

Action

  • Bits 0 to 4 (mask is 0x007F) contain the shirt number. It is 0 if the action is for the team, a shirt number from 1 (0x01) to 25 (0x19), the value 30 (0x1E) for a libero substitution, the value 31 (0x1F), for a player substitution.
  • Bits 8 to 10 (mask is 0x0700) contain the skill, see the table below for the values.
  • Bits 12 to 14 (mask is 0x7000) contain the note, see the table below for the values.

Libero substitution

  • Bits 0 to 4 (mask is 0x001F) contain 30 (0x1E).
  • Bits 5 to 9 (mask is 0x03E0) contain the shirt number of the libero entering the court.
  • Bits 10 to 14 (mask is 0x7C00) contain the shirt number of the libero leaving the court.

  • Bits 5 to 9 (mask is 0x03E0) contain the shirt number of the player entering the court.
  • Bits 10 to 14 (mask is 0x7C00) contain the shirt number of the player leaving the court.
 Action (shirt number up to 99)

An action is defined into four 8-bits integers. The value is splitted into three parts and the meaning of these parts depend on the content.

  • Byte 2 contain the skill, see the table below for the values.

Libero substitution

A libero substitution is coded with four 8-bits values:

  • Byte 1 contains the special code: 250 (0xFA).
  • Byte 2 contains the shirt number of the libero entering the court.
  • Byte 3 contains the contain the shirt number of the libero leaving the court.
  • Byte 4 is 0.

Player substitution

A player substitution is coded with four 8-bits values:

  • Byte 1 contains the special code: 253 (0xFD).
  • Byte 2 contains the shirt number of the player entering the court.
  • Byte 3 contains the contain the shirt number of the player leaving the court.
  • Byte 4 is 0.

 String

A String is saved as the string length followed by the chars in 8-bits ASCII format.

The string length can be 1, 2 or 4 bytes:

  • If the first byte has its bit 8 reset, the length is in the first byte.
  • If the first byte has its bit 8 set and bit 7 reset, the length is coded in 2 bytes.
    The length can be calculated as 256 * (Byte1 && 0x3F) + Byte2.
  • If the first byte has not its bits 7 and 8 set, the length is coded in 4 bytes.
    The length can be calculated as 16777216 * (Byte1 && 0x3F) + 65536 * Byte2 + 256 * Byte3 + Byte4.
