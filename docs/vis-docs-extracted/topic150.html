Filter for press release texts

Filter for press release texts.

 Attributes

The following table lists the attributes for the filter for press release texts.

Name Type Description
Accepted Boolean Accepted state for the press release texts.
The meaning of the values is:
Value Description

Empty
OR
Not specified

 The web service will return all the press releases, not taking account of the accepted state of their texts.
True The web service will return only the press release whose at least one of their text is accepted.
False The web service will return only the press release whose at least one of their texts is not accepted.

If this attribute is specified, the Languages property of the returned press releases will contain only the languages for the texts that are accepted or not, depending of the value of this attribute.
AcceptedOnly Boolean Return only accepted press release texts.
If it is not specified, the returned press releases will contain only the accepted ones.
This attribute is obsolete and is accepted only for compatibility with old client applications.
Please use the Accepted attribute instead.
Categories PressReleaseCategory Categories for the press release.
Each category must be specified by its name or value, and, if more than one category is specified, they must be separated with spaces.
If it is not specified, all categories are included in the returned press release texts.
Category PressReleaseCategory Category of the press release.
If it is not specified, all categories are included in the returned press releases.
This attribute is obsolete and is accepted only for compatibility with old client applications.
Please use the Categories attribute instead.
Language String Language of the press release text.
All press releases are in English and some of them are translated to other languages.
See the description of the Language field of the PressReleaseText data type for a list of valid values.
If it is not specified, the returned texts are not filtered by this criteria and the result can contain more than one text for a press release, in different languages.
PublishOn PressReleasePublishOn Where the press release should be published.
You can specify more than one value, if you separate them with a space. If there is more than one value, the response will contain all the press releases that are published on one of the specified location. For example, if you specify PublishOn="Volleyball Beach", the response will contain all the press releases that are intended to be published on the Volleyball or Beach Volleyball pages.
If it is not specified, the returned texts are not filtered by this criteria.
SpecialType PressReleaseSpecialType List of special press release types to include in the result.
You can specify more than one value, if you separate them with a space. If there is more than one value, the response will contain all the press releases that are of one of the specified special type. For example, if you specify SpecialType="MatchFlash MatchPreview", the response will contain all press releases that are a match flash information or a match preview.
If it is not specified, the returned texts are not filtered by this criteria.
This attribute is obsolete and is accepted only for compatibility with old client applications.
Please use the SpecialTypes attribute instead.
SpecialTypes PressReleaseSpecialType Types for the press release texts.
Each type must be specified by its name or value, and, if more than one type is specified, they must be separated with spaces.
If it is not specified, the returned texts are not filtered by this criteria.
TournamentCodes String Code of the tournaments related to the press release.
You can specify more than one code, if you separate them with a space. If there is more than one value, the response will contain all the press releases that is in relation to one of the specified tournament.
If it is not specified, the returned texts are not filtered by tournament code.
