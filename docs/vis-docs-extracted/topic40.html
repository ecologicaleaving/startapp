Tag
This documentation is preliminary, as the tags API is currently in development. The information here is subject to changes.

Represents a tag.
This can be applied to an article or an image.
A tag contains only data that is not tied to a language. Each article can contain one or more texts, available with the Texts relation.

The name of the XML element for a tag is <Tag>.
The name of the XML element for a list of tags is <Tags>.

This page contains the following information:

  • Names of tags
  • Properties
  • Relations
  • Types of tags
 Properties

The following table lists all the properties for a tag.

Name Type Description
ItemType DataType The type of the tag item.
The value is empty if this is not an item tag.
No Int32 The number.
Text String The text, in the English language.
UserText String The text, in the user language.
User language processing is currently not implemented, so this property has the same value as Text.
Version Int32 The version of the tag.
 Relations

The following table lists the relations for a tag.

Name Type Description
Items Array of TagIten The items linked to the tag.
Currently not implemented.
Texts Array of TagText The texts.
 Types of tags

There are 3 types of tags: item tags, system tags and text tags.

Item tags

An item tag is used when the tag reference an item, for example a player. In this case, the tag assignment contains also the number of the item. When the tag is displayed, the number of the item is separated with a semi-colon: player:100002.

System tags

System tags are represented by a text starting by a #. They are used to mark some special types of an article, for example those for the Trophy Tour with the #trophy-tour tag. You can find a list of the currently defined system tags at the System Tags page.

Text tags

Text tag can be defined by the author of an article or an image. They can be any text.

 Names of tags

Each tag is identified by its number. The name that is displayed to the user can be translated to many languages, as specified by the TagText data type.

A name can contain letters, numbers and the dash char. The dash char is used to separate the words in the name; it cannot be used at the start of the tag. Letters are treated in a case-insensitive way and cannot contain any accented char. The web service converts all texts to lowercase and removes the accents and the other diacritics on the chars.
Examples of names are 2020-olympic-games, jeux-olympiques-2020 (French version of the previous one), world-league and snow-volleyball.

System tags starts with a # char, that is forbidden for other tags.
Examples of system tags are #trophy-tour and #features.
You can find a list of the currently defined system tags at the System Tags page.
