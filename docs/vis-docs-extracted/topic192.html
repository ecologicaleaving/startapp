Request to get the ranking of a volleyball pool
 Request

The syntax for this request is:

Web service request
Copy Code 
<Request Type="GetVolleyPoolRanking"
         NoPool="<pool number>"
         Fields="<list of the fields to return>" />

Where:

Attribute Description
NoPool The number of the volleyball pool.
This is a mandatory parameter.

Fields

The list of fields is mandatory.
It can contain all the fields in the VolleyPollRankingEntry data. All the fields are public, so you can request any one.

 Properties by match format

Matches in a pool can be played in 5 sets or in 7 sets. You can get the number of sets by looking at the format of a match in the pool. There are two sets of properties for the matches by result, one for matches in 5 sets and one for matches in 7 sets. If you need this information, you must get the right properties, depending on the matches format.

Matches in 5 sets

You must use the following properties:

  • MatchesAt03
  • MatchesAt13
  • MatchesAt23
  • MatchesAt30
  • MatchesAt31
  • MatchesAt32

Matches in 7 sets

You must use the following properties:

  • MatchesAt04
  • MatchesAt14
  • MatchesAt24
  • MatchesAt34
  • MatchesAt40
  • MatchesAt41
  • MatchesAt42
  • MatchesAt43
 Rank and Position

The Rank property con be negative for tied teams: the first team has a positive value and the tied ones have a negative value of the same absolute value. To display the rank correctly, you can use the RankText property, that is empty for tied teams. If the ranking you're displaying can be filtered or grouped, you can use the RankTextWithRepeat property, that has always a rank.

As the Rank property can be negative, you must not use it for sorting, instead, you must use the Position property, that always starts at 1 and is incremented for each team.

 Points and sets ratios

When a team has not played any match, its ratios cannot be calculated and should be displayed as an empty string. The value in the ratios properties is -2.

When a team has played matches, but has not lost any point or set, its corresponding ratio is infinite and should be displayed as <MAX>. The value in the corresponding ratio property is -1.

You should use the PointsRatioText and SetsRatioText property to display the ratios correctly.

 Response

When the request is successful, the service will return a <VolleyballPoolRanking> XML element that contains volleyball pool ranking entries.

The response is sorted by Position, so it is returned in the order in which it must be displayed. So there is no need to get the Position property and to sort the ranking in the client application. The Position property is only needed if your application enables the ranking to be sorted by other criteria.

 Errors

In addition to global errors, the following errors can be sent by the server:

Id. Code Description

1002

 BadParameter The No parameter is not a valid 32-bits integer number.

18000

 BadPoolRankingType The ranking method doesn't specify how to rank the pool.

1005

 NoData The volleyball tournament with the specified number does not exist or has been deleted.

18001

 NotARankingPool The specified pool is not a ranking pool.
Only the PlayOff and Ranking pools have a ranking.

1009

 ParameterMissing You must specify the Fields and NoPool parameters.

18002

 UndefinedRankingMethod The ranking method is not defined in the pool.
 Security

This is a public request: any client can retrieve information.

All the fields for a volleyball pool ranking are public.

 Example

Get the ranking for the pool with the number 2188.

Web service request
Copy Code 
<Request Type="GetVolleyPoolRanking"
         NoPool="2188"
         Fields="Position Rank TeamCode TeamName MatchesAt03 MatchesAt13 MatchesAt23 MatchesAt30 MatchesAt31 MatchesAt32 MatchesLost MatchesWon MatchPoints PointsLost PointsWon SetsLost SetsWon" />
