VIS web service documentation
General errors (XML)
 Communication errors

When there is a communication error (HTTP error 404), it means that the web service is not accessible for a moment. It could be because of Internet error, the server is restarted, or any other cause. In this case, the request can be sent later, when the web service is accessible again. Because the username and password are sent for each request, the web service can automatically log in the user and no data is lost.

 Types of errors

There are two types of errors: global errors and request error. When a global error is encountered, no request is processed. When a request error is encountered, only the request with the error is not processed (in the case of multiple request format), the other requests are processed and can return a result or an error.

Single request format

The error is indicated by an HTTP status code and the content of the response is an XML element with the identifier of the error, its code and more information if available.

Global errors and request errors are returned with the same method.

Multiple request format

The errors are indicated by an XML element with the identifier of the error, its code and more information if available.

Global errors are returned as the root element of the response.

Request errors are returned as the response element for the request.

 Web service errors

When there is a web service error, the error message ise sent back to the client. The error message consist of a single <Error> element containing the text of the error.

If the error is raised during the processing of a single request, the <Error> element is the answer for the request. If the error is global to the processing of the list of requests, the <Error> element is the only answer that the client will receive.

In any case, when encountering a web service error, please contact the FIVB with the request and the error. So we can look at the reason and correct the web service.

 Global errors

The following table lists the global errors.

Element Id Description
<AccessTokenExpired>

1

 The access token has expired.
<AccessTokenInvalid>

2

 The access token is invalid.
<AccessTokenRevoked>

3

 The access token has been revoked.
<BadRequestSyntax>

4

 The XML of the request is not well-formed.
<BadUser>

6

 The username or password is not valid.
<Error>

5

 Other server error.
The error message gives more information.
This error indicates an error in the server code and should be sent back, with the request that generated the error, to the FIVB for further investigation.
<RefreshTokenExpired>

7

 The refresh token has expired.
<RefreshTokenInvalid>

8

 The refresh token is invalid.
<RefreshTokenRevoked>

9

 The refresh token has been revoked.
<UnsupportedAuthentication>

10

 The specified authentication method is not supported.
See the Authentication documentation for more information.
<UserIsDeleted>

12

 The user cannot send requests because he/she is disabled.
<UserIsRevoked>

13

 The user cannot send requests because he/she has been revoked.
The error message gives the reason for which the user has been revoked.
 Request errors

The following table lists the common request errors.

Element Id Description
<AccessDenied> 1000 You don't have access to this request.
<BadFilterParameter> 1001 One of the parameters of the filter is not valid.
The error message gives the name of the parameter.
See the documentation of the filter for its parameters.
<BadParameter> 1002 One of the parameters of the request is not valid.
The error message gives more information about the parameter.
See the documentation of the request for exact parameter information.
<BadSortProperty>

1014

 The specified property is not available for sorting.
The error message gives the name of the property.
The properties that can be used for sorting are limited for each data type. See the documentation of the request to know which properties are available for sorting.
If you really need to use this property for sorting, please contact the FIVB to request the use of this property.
<Error> 1004 Other server error.
The error message gives more information.
This error indicates an error in the server code and should be sent back, with the request that generated the error, to the FIVB for further investigation.
<FilterParameterMissing> 1003 A mandatory parameter of the filter is missing.
The error message gives the name of the parameter.
See the documentation of the filter for its parameters.
<InvalidLanguage> 1019 The specified languages have not a valid syntax.
A language specification must use the syntax of the HTTP Accept-Language header.
<InvalidTag> 1017 A tag cannot be parsed.
This error is flagged for an unknown system tag, a bad item tag or a free-text tag that contains invalid chars.
Look at tag syntax for more information about the syntax of the tags.
<MandatoryPropertyMissing> 1013 One or more mandatory properties of an item are missing.
The error message contains the names of the missing properties, separated by a comma.
See the documentation of the item type for the mandatory properties.
<NoData> 1005 There is no data to send back.
This response is used for single-data request, when the parameters of the request matches no data.
<NoLanguage> 1018 A request needs language information and the language was not specified.
Some request needs language information. For example, the Text relation of an article.
<NotImplemented> 1006 A function of the request is currently not implemented.
The error message gives more information about the not implemented function.
If you really need to use a non-implemented function, please contact the FIVB for information about when this function will be implemented.
<NotInNewFormat> 1008 The request is currently not available in the new format (single request format).
You must enclose the request with <Requests></Requests>.
You can find the list of request available in the new format here.
<OK> The request has been executed correctly.
This response is used for requests that doesn't send data back or that just return a single value.
<ParameterMissing> 1009 One of the mandatory parameter of the request is missing.
The error message gives more information about the parameter.
See the documentation of the request for exact parameter information.
<RelationParameterMissing> 1011 A mandatory parameter of a relation is missing.
The error message contains the name of the relation and the name of the parameter.
See the documentation of the request for information about its relations.
<RelationsNotSupported> 1010 The request doesn't support relations.
See the documentation of the request for information about its relations.
<SortsNotSupported>

1015

 The request doesn't support sorting.
Currently the SortBy parameter is only supported for some data type. See the documentation of the request to know whether the sort are supported.
<UnknownRelationName> 1012 A relation is unknown for the request.
The error message contains the name of the relation.
See the documentation of the request for information about its relations.
<VerifyError> 1016 The item has errors during verification.
When saving an item, the item is checked against many criteria. If one or many criteria are not met, the error is returned by the web service with the result of the verification.


In addition to these common errors, each request can return dedicated error. See the documentation of the requests for the errors that can be returned.
